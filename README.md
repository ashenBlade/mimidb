# mimidb

## Описание

Особенности:

1. MVCC с использованием CSN (аналог оракловского SCN):
   - Меньше contention в отличие от снапшота в pg
2. UNDO лог
   - Не нужен VACUUM
   - Поэтому операции SELECT и т.д. будут быстрее
   - Храним дельту изменений (применяем, чтобы восстановить версию старую)
3. Logical ID для кортежей. Отдельный mapping layer - logical id -> physical id (page+idx)
   - Лучше поддержка индексов
   - Хранится как отдельная структура (файл), поэтому файл основных данных может быть любым, т.е. обычный heap, а можно кластеризованный индекс (выбор за пользователем)

## Цели первой версии

Самая тупая СУБД, не для прода:

1. Хранения на диске нет. Пока только в памяти все.
2. SQL и планировщика нет. В первых версиях все буду делать руками.
3. Только crud: SELECT, INSERT, UPDATE, DELETE. Без сложных подзапросов и т.д.
4. Системный каталог только в памяти.

Главная задача - разобраться с ядром: MVCC и конкурентность.
